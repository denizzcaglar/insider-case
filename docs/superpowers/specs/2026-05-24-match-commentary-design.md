# Match Commentary — Design

Status: approved, ready for implementation plan.
Date: 2026-05-24.

## Goal

Each played fixture in the Match Results panel gains a `Commentary` link. Clicking it expands a short narrative summary of the match generated by an LLM (Google Gemini). Results are cached per fixture so the LLM is called at most once per played score for a given fixture.

## User-visible flow

1. The user opens the league page and plays at least one week. Each played fixture row in the Match Results panel shows a small `Commentary` link next to the existing `edit` link.
2. Clicking `Commentary`:
   - If a stored commentary exists for that fixture and its current score, the text expands instantly below the row.
   - Otherwise, the link is replaced with a `Generating…` spinner. The browser calls the new endpoint, which calls Gemini and persists the result. Once it returns (about 1 to 2 seconds), the text expands.
3. While expanded, the link reads `Hide commentary`. Clicking it again collapses the text without re-fetching.
4. If Gemini fails, the user sees an inline error message and the link returns to its initial state. Nothing is cached. The next click retries.

## Invalidation

- When the user edits a fixture result via `PATCH /api/fixtures/{fixture}`, `LeagueService::editResult()` deletes the matching `match_commentaries` row before returning. The next view regenerates against the new score.
- When the user resets the league, `LeagueService::reset()` deletes fixtures. The `match_commentaries.fixture_id` foreign key has `ON DELETE CASCADE`, so cleanup happens at the DB level.

## Architecture

### One concrete class, no interface

```
app/Services/Commentary/
  CommentaryGenerator.php       // calls Gemini, returns 2-sentence text
  MatchCommentaryService.php    // orchestrates cache + generation + storage
app/Http/Controllers/Api/
  MatchCommentaryController.php // GET /api/fixtures/{fixture}/commentary
app/Models/
  MatchCommentary.php           // Eloquent model for match_commentaries table
```

Rationale: `CommentaryGenerator` has a single responsibility (generate commentary text from inputs). No interface is introduced because there is no credible second implementation; tests use `Http::fake()` to stub the network call, which does not require a separate fake class. This matches the YAGNI call that already removed the `TeamDataProvider` seam.

### Class shapes

```php
final class CommentaryGenerator
{
    public function generate(
        Fixture $fixture,
        StandingsTable $before,
        StandingsTable $after,
    ): string;
}

final class MatchCommentaryService
{
    public function __construct(
        private readonly CommentaryGenerator $generator,
        private readonly StandingsCalculator $standingsCalculator,
    ) {}

    public function for(Fixture $fixture): string; // cache-or-generate
}
```

### Service-provider bindings

None needed. Both classes auto-resolve via Laravel's container.

## Endpoint

`GET /api/fixtures/{fixture}/commentary`

Responses:

- `200 { "commentary": "…" }` — happy path.
- `404` — fixture not found (Route Model Binding handles this).
- `422` — fixture exists but is unplayed.
- `503 { "message": "Commentary service not configured." }` — `GEMINI_API_KEY` missing.
- `503 { "message": "Commentary service unavailable; please try again." }` — Gemini call failed.

The endpoint is intentionally not under `Admin/`. It is open like every other endpoint on the API.

## Data model

New table: `match_commentaries`.

```sql
CREATE TABLE match_commentaries (
    id          BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    fixture_id  BIGINT UNSIGNED NOT NULL,
    home_goals  SMALLINT UNSIGNED NOT NULL,
    away_goals  SMALLINT UNSIGNED NOT NULL,
    content     TEXT NOT NULL,
    created_at  TIMESTAMP NULL,
    updated_at  TIMESTAMP NULL,
    UNIQUE KEY match_commentaries_fixture_unique (fixture_id),
    CONSTRAINT match_commentaries_fixture_fk
        FOREIGN KEY (fixture_id) REFERENCES fixtures(id) ON DELETE CASCADE
);
```

Notes:
- `UNIQUE` on `fixture_id` enforces at most one commentary per fixture.
- `home_goals`/`away_goals` are stored for traceability and to make the staleness check observable in the database.
- `ON DELETE CASCADE` handles cleanup when fixtures are deleted on reset.

## Prompt

Built inside `CommentaryGenerator::generate()`.

```
You are a Premier League football commentator. Generate a 2-sentence summary
of the following match. Be specific to the teams and the result; do not use
filler phrases.

Match: {home_name} {home_goals} - {away_goals} {away_name}
Week: {week} of 6 in the Insider League

Before this match:
- {home_name}: {points_before} points (position {position_before})
- {away_name}: {points_before} points (position {position_before})

After this match:
- {home_name}: {points_after} points (position {position_after})
- {away_name}: {points_after} points (position {position_after})

Respond with exactly 2 sentences. No greetings, no signature, no markdown.
```

Standings-before is computed by `StandingsCalculator` over all played fixtures of the season ordered by `(week, id)` and taken strictly before the target fixture. Standings-after includes the target fixture.

## Gemini call

- Method: `POST`
- URL: `https://generativelanguage.googleapis.com/v1beta/models/{model}:generateContent?key={api_key}`
- Default model: `gemini-2.5-flash` (free-tier eligible; overridable via `GEMINI_MODEL` env, e.g. `gemini-2.5-pro` for paid projects)
- Body:

```json
{
  "contents": [{ "parts": [{ "text": "<prompt>" }] }],
  "generationConfig": { "temperature": 0.85, "maxOutputTokens": 100 }
}
```

- Timeout: 10 seconds (`Http::timeout(10)`).
- Successful response parsing reaches `candidates[0].content.parts[0].text`. Anything else is treated as failure.
- Failure handling: log via `Log::warning(...)`, return 503 from the controller, do not write to the cache.

## Configuration

New file `config/commentary.php`:

```php
return [
    'api_key' => env('GEMINI_API_KEY'),
    'model'   => env('GEMINI_MODEL', 'gemini-2.5-flash'),
];
```

`.env.example` gains:

```
# Google Gemini API key (https://aistudio.google.com/app/apikey).
# Required for the per-fixture commentary endpoint.
GEMINI_API_KEY=
```

Service code reads `config('commentary.api_key')`, never `env()` directly.

## Frontend

In `resources/views/league.blade.php`:

- The `fixtureMarkup()` helper for played fixtures gains a second link `Commentary` next to the existing `edit` link.
- New handler `openCommentary(fixtureId)`:
  - If already expanded, collapse and return.
  - Otherwise, set link text to `Generating…`, call the new endpoint, on success insert the text below the row and switch the link to `Hide commentary`. On error, flash a small message and revert.
- The link only appears on played fixtures.

Tiny, no new dependencies. The minimal-build-step principle holds.

## Testing strategy

Three test files, all using `Http::fake()` where the Gemini network call is involved.

### `tests/Unit/Services/Commentary/CommentaryGeneratorTest.php`

- Builds a fake `Fixture`, `StandingsTable` before, `StandingsTable` after, calls `generate()`.
- Uses `Http::fake([...])` returning a canned `candidates[0].content.parts[0].text`.
- Asserts:
  - The request URL contains the model and key (key from `config('commentary.api_key')`).
  - The request body contains the team names, the score, the week number, and the before/after standings lines.
  - The returned string matches the canned text.
  - On non-200 response, throws a `CommentaryGenerationException`.
  - On missing `candidates`, throws.

### `tests/Feature/Services/Commentary/MatchCommentaryServiceTest.php`

- Uses `Http::fake()` with a counter so we can assert how many calls happened.
- Tests:
  - First call to `for($fixture)` triggers one HTTP call and persists a row.
  - Second call with the same fixture and unchanged score hits cache and makes zero HTTP calls.
  - After `editResult($fixture, ...)`, the next `for($fixture)` triggers a new HTTP call (cache busted).
  - On unplayed fixture, throws.

### `tests/Feature/Http/MatchCommentaryEndpointTest.php`

- Full HTTP integration. Uses `Http::fake()` for Gemini.
- Asserts:
  - `GET /api/fixtures/{played}/commentary` returns 200 with `commentary` field.
  - `GET /api/fixtures/{unplayed}/commentary` returns 422.
  - `GET /api/fixtures/9999/commentary` returns 404.
  - With `GEMINI_API_KEY` unset, returns 503 with the configured message.
  - With Gemini returning 500, the endpoint returns 503 and no row was persisted.

## Out of scope

- A separate `FakeCommentaryGenerator` class (not added; `Http::fake()` is sufficient).
- A `CommentaryGenerator` interface (not added; no second implementation).
- Streaming commentary mid-match (the SSE tick engine lives on a separate branch).
- Multi-provider support (Gemini today; swap by editing the generator if and when needed).
- Background-job generation (kept synchronous, lazy + cached).
